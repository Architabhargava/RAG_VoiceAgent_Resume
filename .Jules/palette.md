## 2025-05-14 - Robust Voice Interaction State Management
**Learning:** In voice-enabled interfaces using asynchronous APIs like `getUserMedia`, a simple boolean flag or relying solely on the `MediaRecorder.state` is insufficient. Users may release the "Push to Talk" button before the microphone permission is granted or the recorder is initialized, leading to "hot mic" race conditions where the recording starts but the UI indicates it has stopped.
**Action:** Implement an `isRecordingIntent` flag that tracks the user's *desire* to record. Check this flag immediately after the asynchronous permission request resolves; if the user has already released the button, stop the tracks and abort the recording process. Synchronize all visual updates, keyboard events, and mouse/touch events to this intent flag.
