## 2024-04-13 - "Push to Talk" Accessibility and UX
**Learning:** Implementing "Push to Talk" requires synchronizing multiple interaction modes (mouse, touch, keyboard) and providing immediate, multi-modal feedback. Standard CSS properties like `box-shadow` for focus rings and `transform` for tactile feedback are more reliable and accessible than non-standard utility-like properties in a vanilla environment. Keyboard listeners should be global but target-aware to prevent interference with other inputs. Keep CSS/JS minimal to adhere to micro-UX constraints while maintaining high accessibility standards.
**Action:** Always use valid vanilla CSS for focus indicators (avoiding framework-specific shorthand like `ring`). Ensure `aria-pressed`, `aria-live`, and dynamic text are updated together to maintain an accessible state for screen readers. Always clean up MediaStream tracks and revoke object URLs to prevent leaks and hardware locking. Reverting to explicit local hostnames for WebSockets improves local dev compatibility when served across different ports.
