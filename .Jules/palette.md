## 2025-05-15 - [Improving Voice Interaction with Centralized State and Multi-modal Support]
**Learning:** For "Push to Talk" (PTT) interfaces, visual feedback (color/scale changes) and textual updates (e.g., 'Release to Send') must be immediate and synchronized with interaction states (Mouse, Keyboard, Touch). Relying only on `mousedown`/`mouseup` can lead to "hot mic" or "sticky mic" issues if the cursor leaves the button. Centralizing the `start` and `stop` logic around an `isRecording` flag, and adding a `mouseleave` listener, significantly improves interaction safety and reliability.
**Action:** Always implement a centralized `start`/`stop` function and a safety state flag for PTT buttons to handle diverse user behaviors (like dragging off the button) and input methods (Keyboard Space bar, Touch).
